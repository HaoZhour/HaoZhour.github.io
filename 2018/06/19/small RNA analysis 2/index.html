mirdeep2 的使用

Posted on 2018-06-19 | In Small RNA-seq data analysis

mirdeep2是分析已知成熟miRNA和新miRNA的工具

数据

  • cel_cluster.fa: 参考基因组
  • mature_ref_this_species.fa:这个物种的成熟miRNA
  • mature_ref_other_species.fa: 其他物种的成熟miRNA（最好是相近的物种）
  • precursors_ref_this_species.fa:这个物种的hairbin 序列
  • reads.fa: 测序的去除重复序列的unqiue序列。

步骤

Step 1:去除重复序列

note：

  • bos是要加上的物种，官网主页上的documentation 还没有更新这里要加物种，不加会报错。
  • 要用它的自带的程序 collapse_reads_md.pl 去除，因为要求reads的header 符合格式如下：

    Collapses reads in the fasta file to make each sequence entry unique. Each collapsed
    entry will have an identifier that follows an underscore ‘_’ separated format. Example:
    mmu_1189273_x10

  • 我的数据是用其他程序collapse的，header只有reads数目10，本想手动改成seq_x10，结果报错说必须要有这三部分，并且每一个header还要是unique。

note：

  • 这里所有输入的fa数据的header，还有arf格式的数据不能有空格，字段之间要求以_链接。看了一些还提供几个脚本检查数据是否合格。不合格的还有脚本进行转化。
  • 其他的还有一些脚本进行各种常用数据格式的转化，利用起来不用再写类似的脚本啦

可能的报错

  • mirdeep is not in arf format 问题是reads header格式不对，严格参考这个格式mmu_1189273_x10
  • mirdeep does not start with ‘>identifier’ 问题是reads header格式不对，严格参考这个格式mmu_1189273_x10
  • .fa has not allowed whitespaces in its first identifier 问题是输入文件有空格存在，替换成_ 即可。
